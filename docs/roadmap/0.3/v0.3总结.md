# v0.3 版本总结

## 目标回顾

0.3 的主目标是提高日常易用性、减少频繁切到 CLI 的次数，并降低新环境 setup 成本。

核心方向：

- 常用工作流尽量留在 HaloCreek 应用内完成，包括发 prompt、看 session 回复、判断 session 是否完成、进入 Review。
- 前台 session 从“终端前台”调整为“应用内前台”，把发送目标和状态展示目标绑定到应用状态。
- 从 Codex session JSONL 读取消息、任务状态和 token 信息。
- 引入文件系统监听，减少单纯依赖定时刷新。
- 尝试 Windows Codex + psmux 路线，降低 WSL/tmux 依赖。
- 提供依赖安装脚本。

## 已完成

### 应用内 session 体验

- Launch / Resume 后不再自动打开 Windows Terminal。
- Ongoing Sessions 双击只切换应用内 front，不再切换终端 front。
- `SendToFront` 发送到当前应用内 front session。
- Ongoing Sessions 增加显式 `CLI` 入口，需要终端时才打开 Windows Terminal 并 attach 到对应 psmux session。
- 移除 `Ctrl+Alt+T` 作为常规终端入口；保留 `Ctrl+Alt+H` 打开悬浮 prompt 输入框。

### Session 状态读取与展示

- 通过 Codex session JSONL 读取用户消息、assistant 消息、任务状态、更新时间和 token 信息。
- Prompt Editor 右侧增加当前 front session 的状态视图。
- assistant 消息接入 Markdown 渲染，支持代码块、链接和基本文本格式。
- 本地文件链接支持默认打开；带行号链接可通过 `MarkdownLineJumpCommands` 配置跳到 VS Code、Rider 或 Visual Studio。
- session 完成后在应用未聚焦时发送 Windows 通知；点击通知会激活应用并切换对应 session 为 front。

### Snapshot 响应性

- `WorkspaceSnapshotStore` 接入 `FileSystemWatcher`。
- snapshot 读取后可发布监听路径；文件变化经过 debounce 后触发刷新。
- 仍保留定时刷新作为兜底。

### psmux / Windows Codex 路线

- 后台 session 管理从 WSL tmux 路线切到 Windows 侧 `psmux.exe`。
- Codex 启动在 Windows 侧 psmux session 中执行，默认可执行文件仍为 `codex`。
- `scripts\install_deps.bat` / `scripts\install_deps.ps1` 提供 psmux、Codex CLI、Git for Windows 和 TortoiseGit 的安装与验证入口。

### Dead / Attention / Restart

- 增加 keep-alive snapshot，用 `psmux capture-pane` 探测 session 是否还存在，以及是否能看到 Codex 输入面板。
- UI 展示 `Dead` 和 `CLI attention` 状态。
- `Dead` session 支持 `Restart`：从原始 prompt 启动，或对已知 Codex session id 执行 resume。

## 调整和降级

- Review 页当前保留 Modified + Unreviewed / Reviewed 两块；旧的右侧 Review Prompt 和 ClipLocate UI 不再作为 0.3 用户入口。
- `ReviewClipboardContextService` 仍保留剪贴板定位能力，但当前没有业务 UI 消费；后续确认真实入口后再重新接入。
- `scripts\check_deps.bat` 仍是旧 WSL/tmux 检查逻辑；0.3 推荐使用 `scripts\install_deps.bat`。
- 应用内 Codex 登录和更新入口没有在 0.3 完成，仍需要依赖 Codex CLI 自身能力或外部命令完成。

## 后续建议

- 把 `check_deps.ps1` 更新到 Windows/psmux 依赖集合，避免与 `install_deps.ps1` 分叉。
- 清理仍保留的 WSL/tmux 命名和兼容代码，尤其是 `TmuxService` 等历史命名。
- 重新评估 ClipLocate 的用户入口：如果 Review 工作流确实需要，再设计明确 UI；否则删除保留能力。
- 设计 Codex 登录/更新入口，解决 setup 过程中仍需要离开应用的问题。

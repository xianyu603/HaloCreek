目标: 进一步提高易用性，降低setup成本.
易用性:
	常用工作流尽量不要切到cli(发prompt 看回复, review...) 减少现在需要频繁在cli 应用切换, 复制体验也不好. tmux仍然作为后台, 并保留打开cli的能力, 偶尔有用.
	前台session变'软', 切换前台只切换展示UI, 只有在打开cli时才进行比较重的切换操作, 并且应当低频(设计上是开一会就关了)
	session状态, 消息之类的从jsonl读取. 读到完成发通知; prompt input view 里面新增一个只读文本框, 里面写上'当前session的最近汇报' 实时刷新
	对应的基础能力: 文件系统监听, snapshot考虑也接一下(或者当前session扫描可以接到snapshot里面)
	通知二选一, 快捷进review tab或快捷弹悬浮框, 如果做不到快捷弹悬浮框, 

setup:
	减少对wsl的依赖, 尝试使用win codex和win psmux 作为后台
	提供一键依赖安装脚本.
	解决登录/更新无法在应用内完成的问题

明确暂不进行: 多session的正式并行体验, 仍然放到'并行'主题. send reply仍然智能发送给前台session.

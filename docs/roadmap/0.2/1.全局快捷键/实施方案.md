# 全局快捷键实施方案

## 目标

应用启动后注册两个写死的全局快捷键，降低窗口切换成本：

1. 唤起 HaloCreek 主窗口并切到前台。
2. 唤起当前 front terminal/session；如果没有可用 terminal/session，则给出已有的前台 session 失败提示。

快捷键配置先不接入配置文件，后续再扩展为可配置。

## 类与位置

新增 `HaloCreek/Infrastructure/GlobalHotkeyRegistrar.cs`。

职责：

- 注册默认全局快捷键。
- 释放快捷键注册和平台资源。
- 将快捷键触发转换为应用内部事件。
- 不处理业务动作，不依赖 ViewModel。

新增 `HaloCreek/Services/GlobalHotkeyActionService.cs`。

职责：

- 编排快捷键触发后的业务动作。
- `ActivateMainWindow()`：唤起主窗口到前台。
- `ActivateFrontTerminal()`：调用现有 `SessionLifecycleService.ActivateFrontClient()`。
- 需要 UI 线程执行的动作统一投递到 `Dispatcher.UIThread`。

接口约定：

- `GlobalHotkeyActionService` 暴露普通方法，不使用 `ICommand`；全局快捷键不是 ViewModel 交互命令。
- `GlobalHotkeyRegistrar` 只发布 `MainWindowHotkeyPressed` / `FrontTerminalHotkeyPressed` 事件，不接收业务命令或业务服务。
- `App.axaml.cs` 作为 composition root 负责把 registrar 事件绑定到 action service 方法。

## 接入方式

在 `App.axaml.cs` 的应用组装流程中创建：

1. 先创建 `SessionLifecycleService` 等现有服务。
2. 创建 `GlobalHotkeyActionService`。
3. 创建并启动 `GlobalHotkeyRegistrar`。
4. 将 registrar 放入 `AppDisposeScope`，随应用退出统一释放。

不放入 MVVM。全局快捷键属于应用级入口，不是某个 View 的局部交互；ViewModel 只保留页面状态和页面命令。

## 平台边界

首版按 Windows 桌面环境实现。若注册失败，启动不应静默吞掉：

- 写日志说明哪个快捷键注册失败。

唤起主窗口所需的平台细节优先收敛在 registrar 的平台实现内，业务层只调用语义明确的方法。

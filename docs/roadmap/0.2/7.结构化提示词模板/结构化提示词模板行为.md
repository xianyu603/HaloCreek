# 结构化提示词模板行为

## 目标

结构化提示词模板用于在 prompt 输入框中插入篇幅较长、结构较固定的提示词文本，降低重复组织大段 prompt 的成本。

首版只解决两个入口：

- 预设模板：应用内置一组常用结构化 prompt 模板。
- 最近 Initial Prompt：展示当前 workspace 最近若干个历史 session 的 initial prompt。

这两个入口都通过 prompt 输入框的右键菜单触发。首版不做搜索、不做模板管理界面、不做变量替换，也不接入 `#` 快捷语补全。

## 触发入口

用户在 prompt 输入框中右键时，菜单提供两个单级入口：

```text
Templates
Recent Initial Prompts
```

两个入口首版都只使用一级候选菜单，不再按类别展开二级菜单。

示例：

```text
Templates
  实现功能
  修 Bug
  代码审查
  方案审查

Recent Initial Prompts
  实现提示词编辑器...
  审阅自动补全方案...
```

菜单项点击后把对应完整文本插入到当前 prompt 光标位置。移动鼠标或键盘选择菜单项时只更新预览，不修改 prompt 文本。

## 候选展示

菜单项只展示短信息：

- 标题。
- 可选摘要。
- 最近 initial prompt 直接使用 prompt 摘要。

长模板正文和完整历史 prompt 不直接塞进菜单项，也不依赖 tooltip 展示全文。

### Templates

`Templates` 展示应用内置模板。每个模板包含：

- `title`：菜单展示标题。
- `description`：可选摘要，用于菜单项或预览标题区。
- `insertText`：点击后插入 prompt 输入框的完整正文。

首版模板数量控制在少量高频场景，避免菜单过长。建议初始模板：

- `实现功能`：目标、范围、约束、验收、验证。
- `修 Bug`：现象、期望、复现、相关文件、验证方式。
- `代码审查`：按严重程度列问题，不改代码。
- `方案审查`：审阅方案里的矛盾、风险、遗漏和过度设计。
- `先读代码再给方案`：先定位现有结构，再给可执行计划。
- `小范围重构`：限定行为不变，要求列验证点。
- `补测试/验证`：围绕已有改动补最小验证。
- `提交前检查`：检查 diff、build/test 和残余风险。

首版不为模板提供分类层级。后续如果模板数量明显变多，再考虑把 `Templates` 拆成二级菜单或独立选择器。

### Recent Initial Prompts

`Recent Initial Prompts` 展示当前 workspace 最近 5 个 session 的 initial prompt。

候选来源语义：

- 只读取当前 workspace 对应的历史 session。
- 按 session 最近更新时间或创建时间倒序展示，具体使用哪个时间以现有历史 session 排序能力为准。
- 只展示 initial prompt 非空的 session。
- 完全文本相同的 initial prompt 只展示最新一条。
- 最多展示 5 条。

菜单标题直接使用 initial prompt 摘要：

```text
prompt前若干字符
```

如果 prompt 首行很短，可以继续拼接后续非空文本并压缩空白。标题只服务识别，不影响插入正文。

## 预览框

当用户在 `Templates` 或 `Recent Initial Prompts` 菜单中移动选中项时，输入框附近展示一个稳定的预览框。

预览框用于阅读完整文本，不能触发写入。

基础行为：

- hover 或键盘选中候选项时，预览框展示该候选完整正文。
- 选中项变化时，预览内容同步变化。
- 菜单关闭时，预览框关闭。
- 没有选中项时，不展示预览框，或展示空白预览。
- 预览框可滚动。

预览框不是 tooltip。它应是尺寸稳定的局部弹层，避免长文本 tooltip 超屏、闪烁或无法键盘阅读。

建议尺寸：

- 宽度固定或限制在 420 到 560 px。
- 高度固定或限制在 260 到 360 px。
- 正文支持换行和垂直滚动。

预览框不提供编辑能力。首版也不提供复制、固定、搜索、高亮或 diff。

## 插入行为

点击候选项或按 Enter 接受当前候选时，把候选完整正文插入到 prompt 当前光标位置。

`Templates` 和 `Recent Initial Prompts` 首版使用相同插入语义：

- 不替换整个编辑器内容。
- 不清空已有 prompt。
- 不弹确认框。
- 不自动 launch。
- 不自动 send to front。

插入格式：

- 如果光标前后都已经是空白或文本边界，直接插入正文。
- 如果光标前不是空白，插入前补一个换行。
- 如果光标后不是空白，插入后补一个换行。
- 插入后光标移动到插入正文末尾。

正文内部换行按模板或历史 prompt 原文保留。首版不做变量展开、占位符跳转、缩进修正或 Markdown 结构重排。

## 菜单关闭

以下动作关闭菜单和预览框：

- 点击候选并完成插入。
- 按 Enter 接受候选并完成插入。
- 按 Esc。
- 输入框失焦。
- 右键菜单正常关闭。

关闭菜单不应修改 prompt，除非用户已经显式接受候选。

## 空状态

### Templates

如果内置模板列表为空，不显示 `Templates` action。

### Recent Initial Prompts

如果当前 workspace 没有可用 initial prompt，不显示 `Recent Initial Prompts` action。
## 与现有功能的关系

结构化提示词模板不替代 `#` 快捷语。

- `#` 快捷语继续用于短句、短约束和常用片段。
- 结构化模板用于篇幅较长、有固定段落结构的 prompt。

结构化模板也不替代 History 页面。

- History 页面继续用于查看、恢复 session。
- `Recent Initial Prompts` 只提供最近 5 条 initial prompt 的快速插入入口。

## 非目标

首版不做：

- 模板搜索。
- 模板编辑界面。
- 用户自定义模板文件。
- 模板热更新。
- 模板变量、参数、占位符跳转。
- 模板分类层级。
- 最近 prompt 的整段替换。
- 最近 prompt 的 pin、收藏或删除。
- 与 `#` 自动补全合并。
- 根据 Git diff、剪贴板、workspace 文件动态渲染模板。

这些能力等首版插入体验稳定后再单独设计。

## 风险和降级

- 单级菜单只适合少量模板。模板数量超过 15 到 20 个后，菜单会变长，应重新评估分类或选择器。
- 最近 initial prompt 通常较长，预览框必须可滚动，否则很难确认内容。
- 当前首版统一插入到光标，不提供整段替换。如果实际使用中恢复历史 prompt 的频率更高，再增加 `Replace Editor` 动作。
- 预览框位置可以首版固定在输入框附近，不要求跟随菜单项精确定位；只要不会遮挡当前菜单和主要输入内容即可。

# 快捷键悬浮框实施方案

## 目标

将现有 `Ctrl+Alt+H` 行为从“唤起主窗口”替换为“唤起悬浮 Prompt 输入框”。

不新增全局快捷键，不调整 `Ctrl+Alt+T` 的前台 terminal/session 行为。

## 交互

- 按 `Ctrl+Alt+H` 时显示悬浮 Prompt 输入框；如果已经显示，则激活并聚焦输入框。
- 悬浮框复用 Prompt Editor 中的 `PromptInputView` 主要交互：
  - 输入 prompt。
  - 拖拽路径追加。
  - 自动补全。
  - `Ctrl+Enter` launch。
  - `Alt+Enter` send to front。
- `Esc` 先由悬浮框内部内容按正常输入事件处理；如果事件未被处理，再由悬浮窗口隐藏自身。

## 实现边界

继续保持现有全局快捷键边界：

- `GlobalHotkeyRegistrar` 只负责注册 `Ctrl+Alt+H`，并发布 `MainWindowHotkeyPressed` 事件。
- `GlobalHotkeyActionService` 负责把 `MainWindowHotkeyPressed` 编排到悬浮框显示动作。
- 新增悬浮框相关服务管理窗口和 ViewModel 生命周期。
- `PromptInputViewModel` 不感知自己位于 Prompt Editor 还是悬浮框。

## 类与文件

新增：

- `HaloCreek/Services/FloatingPromptService.cs`
  - 持有并复用一个悬浮窗口实例。
  - 提供 `ShowOrActivate()` 和 `Dispose()`。
  - 创建独立的 `PromptInputViewModel`，注入现有 `SessionLifecycleService`、`AppCommonRuntime`、`CompletionCoordinator`。
- `HaloCreek/ViewModels/FloatingPromptViewModel.cs`
  - 暴露 `PromptInputViewModel PromptInput` 给xaml绑定。
  - 负责释放 `PromptInput`。
- `HaloCreek/Views/FloatingPromptWindow.axaml`
  - 使用轻量窗口承载 `PromptInputView`。
  - 默认 `ShowInTaskbar=false`、`Topmost=true`。
- `HaloCreek/Views/FloatingPromptWindow.axaml.cs`
  - 在内部内容未处理 `Esc` 时隐藏窗口。
  - 显示后聚焦内部输入框。

修改：

- `HaloCreek/Services/GlobalHotkeyActionService.cs`
  - 将 `ActivateMainWindow()` 替换为 `ActivateFloatingPrompt()`, 原有ActivateMainWindow删去。
- `HaloCreek/App.axaml.cs`
  - 在 composition root 创建 `FloatingPromptService`。
  - 将 `MainWindowHotkeyPressed` 绑定到悬浮框显示动作。
  - 将 `FloatingPromptService` 加入 `AppDisposeScope`。

## 复用策略

悬浮框不共享 Prompt Editor 里的同一个 `PromptInputViewModel` 实例，而是新建独立实例。

原因：

- 避免悬浮框和 Prompt Editor 的输入文本互相覆盖。
- 避免补全菜单、caret、selection 等交互状态互相影响。
- 生命周期更清楚，悬浮框关闭或应用退出时只释放自己的输入 VM。

## 首版限制

- 仍然只支持 Windows 全局快捷键；非 Windows 平台沿用现有跳过注册行为。
- 悬浮框位置首版可做屏幕居中，不做鼠标跟随和位置记忆。
- 不引入快捷键配置项。

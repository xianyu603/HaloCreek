# 快捷语补全行为

## 目标

快捷语是 prompt 自动补全的第一个真实来源。它用于把常用提示词片段以短 token 方式快速插入 prompt 输入框，减少重复输入。

首版只定义用户可见行为和配置语义，不要求配置落文件，也不要求提供配置编辑界面。实际快捷语数据可以先写死在应用代码中。

## 触发入口

快捷语使用 `#` 作为触发字符。

用户在 prompt 输入框中输入满足自动补全触发规则的 `#token` 后，候选框展示快捷语候选。`#` 之后的文本称为 query。

示例：

```text
#改
#review
```

快捷语补全复用自动补全框架的既有键盘行为：

- `Down` / `Up` 在当前列表中移动选择。
- `Right` 在当前项有二级菜单时进入二级菜单。
- `Left` 从二级菜单返回上一级。
- `Enter` 接受当前选中项并写回输入框。
- `Esc` 退出当前选择态。

接受候选后，补全框架按既有规则替换当前 `#token`，并在插入文本后追加空格。

## 配置概念

快捷语按两级结构组织：

```text
类别 -> 条目
```

类别用于浏览和精确定位一组快捷语。条目用于最终插入 prompt 文本。

类别包含：

- `name`：类别主名称。
- `aliases`：类别别名列表。
- `description`：可选说明，用于候选列表展示。
- `items`：该类别下的快捷语条目。

条目包含：

- `title`：条目展示名称。
- `aliases`：条目别名列表。
- `description`：可选说明，用于候选列表展示。
- `insertText`：接受该条目后写入 prompt 的完整文本。

首版配置只表达静态文本快捷语。它不表达变量、参数、动态上下文读取、文件引用、条件渲染或多步动作。

## 展示规则

快捷语候选根据 query 分三种展示形态。

### query 为空

当用户只输入 `#`，即 query 为空时，候选框展示所有一级类别。

每个类别可以展开二级菜单。用户可以手动导航到类别下的条目，再接受具体条目。

示例：

```text
输入: #
一级列表:
- 改
- 查
- 审
```

如果用户选中 `改` 并按 `Right`，二级列表展示 `改` 类别下的所有条目。

### query 精确匹配类别

当 query 非空，并且精确匹配某个类别的 `name` 或 `aliases` 时，候选框不展示类别本身，而是直接以一级列表展示该类别下的所有条目。

示例：

```text
输入: #改
匹配类别: name = 改
一级列表:
- 修改代码
- 修改方案
```

这个形态优先服务快速输入。用户已经明确输入类别名时，不需要再多按一次 `Right` 展开类别。

### query 未精确匹配类别

当 query 非空，并且没有精确匹配任何类别时，候选框不展示二级菜单。

此时在所有类别的所有条目中做模糊匹配，并把匹配到的条目作为一级列表展示。

示例：

```text
输入: #代码
一级列表:
- 修改代码
- 解释代码
- 审查代码
```

首版不设置展示上限。所有匹配条目都可以展示。

## 匹配规则

类别精确匹配只匹配类别的 `name` 和 `aliases`。

条目模糊匹配可以匹配：

- 条目的 `title`。
- 条目的 `aliases`。
- 条目的 `description`。
- 条目的 `insertText`。

匹配大小写不敏感。

类别精确匹配优先级高于条目模糊匹配。只要 query 精确匹配到一个类别，就只展示该类别下的条目，不再混入其他类别的模糊匹配结果。

## 冲突规则

类别的 `name` 和 `aliases` 应在快捷语配置内尽量保持全局唯一。

如果多个类别被同一个 query 精确匹配，首版按配置解析顺序保留先解析到的类别，忽略后解析到的重复类别名或别名，并在配置解析时打印 warning。运行时不再合并多个类别，也不因为这类冲突中断快捷语补全。

条目标题和别名不要求全局唯一。多个条目模糊匹配到同一个 query 时，可以全部展示。

## 排序规则

首版排序以配置顺序为准。

query 为空时，类别按配置顺序展示，类别下的条目也按配置顺序展示。

query 精确匹配类别时，该类别下的条目按配置顺序展示。

query 未精确匹配类别时，模糊匹配结果按类别配置顺序和条目配置顺序展开展示。

首版不引入权重、最近使用、拼音排序、分词排序或高亮排序。

## 空状态

当 query 非空且没有精确匹配类别，也没有匹配到任何条目时，候选框按自动补全框架的既有空状态展示。

快捷语来源不额外弹窗，不写入 prompt，也不自动创建新条目。

## 非目标

首版不做：

- 配置文件格式。
- 配置热更新。
- 配置编辑界面。
- 快捷语分组管理 UI。
- 变量替换。
- 参数化模板。
- 动态读取 workspace、Git diff、剪贴板或 session 上下文。
- 最近使用排序。
- 展示数量上限。
- 拼音、分词或语义匹配。

这些能力后续可以在快捷语行为稳定后单独设计。

# 应用状态/配置聚合

本文记录当前 workspace 与配置加载的聚合边界。它解决的是“当前应用状态如何成为唯一事实源，以及 effective config 如何随 workspace 变化而更新”的问题，不解决平台公共操作或统一错误处理入口的问题。

## 定位

应用状态/配置聚合是一个有业务语义的运行状态服务。它代表当前用户选中的工作上下文，以及由该上下文派生出的配置。

命名：

- `WorkspaceRuntimeService`

当前最核心的职责是把 workspace 和 config 放到同一个状态边界里，而不是继续让多个 ViewModel 各自持有 workspace path，并在需要时各自调用 `ConfigService.LoadEffectiveConfig(...)`。

## 应该收口的能力

- 当前 workspace path 的唯一来源。
- workspace 校验和切换入口。
- 最近 workspace 缓存。
- workspace changed 通知。
- 当前 workspace 对应的 effective config。
- config 重加载、缓存和刷新策略。

`ConfigService` 可以在这个设计下退成内部实现细节，只负责从全局配置文件和 workspace 配置文件读取、合并配置。外部对象优先依赖应用状态/配置聚合，而不是直接依赖 `ConfigService`。

## 不应该收口的能力

- 平台公共操作，例如打开目录选择器、启动终端、弹窗。
- 统一错误处理政策。
- Git、tmux、terminal、session lifecycle 等能力服务的业务逻辑。
- 底层 service 的隐式当前 workspace 输入。

尤其是 `TmuxService` 不适合直接感知当前 workspace 或当前 config。它应该继续接收明确参数，例如 workspace path、executable、arguments。这样它仍然只是 tmux 能力边界，而不是“按当前应用状态启动 Codex”的编排服务。

## 依赖原则

应用状态/配置聚合主要面向 ViewModel 和应用编排层。

推荐方向：

```text
ViewModel / orchestration
  -> 读取 WorkspaceRuntime 的当前 workspace/config
  -> 调用 GitService / SessionLifecycleService / TmuxService，并传入明确参数
```

不推荐方向：

```text
GitService / TmuxService / TerminalService
  -> 自己读取 WorkspaceRuntime.CurrentWorkspacePath
```

后者会把当前应用状态变成底层 service 的隐式输入，降低可测试性，也会让这些 service 难以支持“对指定 workspace 执行操作”的场景。

## 与公共服务聚合的关系

应用状态/配置聚合和公共服务聚合是两件独立的事，没有明显的直接依赖关系。

应用状态/配置聚合不需要知道平台公共操作和错误处理政策。它可以只表达状态切换、配置加载和结果。如果 workspace 切换失败是否弹窗、是否写入全局状态，需要由调用方或更高一层编排决定。

公共服务聚合也不需要知道当前 workspace 或 effective config。它只提供应用级基础设施能力。

两者可以被同一个 ViewModel 或编排对象同时使用，但不需要互相引用。

## 长期收益

- 消除 `MainWindowViewModel` 手工向各 tab 分发 workspace 的模式。
- 避免多个 ViewModel 重复保存 workspace path。
- 避免多个 ViewModel 在不同时间各自读取 effective config，导致配置使用时机分散。
- 为后续 config 热加载、workspace 切换一致性、配置错误展示留出明确边界。
- 保持底层能力服务参数显式，不把当前应用状态扩散成全局隐式输入。

## 实施约束

- 当前 workspace 和 effective config 应由同一个状态边界维护。
- `ConfigService` 保留，优先作为内部 reader/merger，不作为大量 ViewModel 的公开依赖。
- workspace/config 聚合不应承担错误展示职责。
- 底层 service 仍应通过方法参数接收 workspace/config 相关输入。
- 命名避免使用 `AppContext`，以免和 `System.AppContext` 混淆。
